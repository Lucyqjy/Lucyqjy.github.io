实验3——利用云平台搭建个人博客

实验目的：本实验旨在通过实践操作，掌握基于云计算环境的网站部署技能。

具体包括：构建环境并实现个人博客/网站的搭建，撰写实验报告，并将这份报告作为一篇博文发布在自己的博客 / 网站上。

1.在github平台注册并登录

2.安装Node.js

登录node.js官网https://nodejs.cn/下载长期支持版本

下载完成后在终端cmd命令行输入node -v和npm -v检查安装是否成功：

由此可以知道，node.js已经安装成功。

3.安装Hexo

在D盘下新建一个Myblog文件夹用于存放发布的博客，在Myblog文件夹中右键单击打开Git Bash，输入npm install -g hexo-cli进行安装，安装完成后输入hexo -v检查安装是否成功：

可以看到，已经成功安装hexo相关环境文件。

4.搭建简单的建设网站

仍然在Myblog文件夹中，打开Bash，输入hexo init初始化网页，成功运行后可以在文件夹中看到多个文件夹以及文件。

【在课上实验的过程中，遇到了一些问题，为了确保能够顺利登录github网站，在实验过程中开启了Steam++带有的网络加速器，导致在hexo init这一步过程中持续出现如下报错：

通过直观感受只看到了”Error:EPERM:operation not permitted,mkdir ‘D:\’ at async object.mkdir(node:internal/fs/promises:857:10)”这一条报错，因此浪费了大量的时间在检查访问权限上，导致没有当堂完成实验并且验收，但是实际上的问题是在上方的白色字体中”SSL certificate problem:unable to get local isser certificate”无法获得本地许可才是出错的根本原因，课下解决此问题的时候关闭了Steam++加速器后即可正常进行hexo init的文件下载】

至此，搭建网站所需的最基本的文件已经下载完成。

在bash中输入hexo g生成静态网页，然后输入hexo s打开本地服务器：

按住Ctrl键单击bash中给出的网址可以浏览博客主页

在bash中输入hexo new “帖子名”发布新帖子。

5.连接本地与github远程服务器

登录github账号，新建仓库名为Lucyqjy.github.io的仓库（为了分配得到域名为http://Lucyqjy.github.io的网址）

在bash中输入git config –global user.name Lucyqjy后回车再输入git config –global user.email qdqiujinyuan@126.com 登录github用户名和邮箱；

输入ssh-keygen -t rsa -C qdqiujinyuan@126.com 生成SSH RSA密钥，回车后在文件夹下C:\Users\qdqiu.ssh可以找到id_rsa.pub文件，复制文件中的rsa密钥添加至github的setting中的SSH and GPG keys中，作为一个新的SSH密钥；

在git bash中输入ssh -T git@github.com，回车可以得到下图，说明本地与github远程服务器连接成功。

6.将本地项目部署至github

修改Myblog文件夹中的_config.yml文件，将最后一个配置网址改为仓库地址：

在bash中输入npm install hexo-deployer-git –save安装工具；

输入hexo d将本地项目推送到github中

Bash中提示信息Deploy done:git表明已经将本地文件推送至github仓库中；

此时打开Lucyqjy.github.io仓库的setting，找到Pages设置（这里需要确保仓库是public属性，private仓库无法进行构建和部署配置）

将Build and deployment部分配置如下图：

配置后刷新页面得到github给出的个人网址，此时即可访问个人网站

至此，个人博客主页的基础搭建和配置就结束了。但是可以注意到，当前网页还非常的（丑陋）简陋，因此可以进行部分博客网页的美化工作。
